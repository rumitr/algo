Mutable vs Immutable Objects

A mutable object can be changed after it's created, and an immutable object can't.

PHP doesn't have any built-in immutable types. Everything is mutable: 

  $array = [4, 9];

$array[0] = 1;
// $array is now [1, 9]
PHP

Other languages have immutable objects though. As an example, in Javascript, you can make objects immutable by freezing them: 

  const array  = [4, 9];

// Make it immutable
Object.freeze(array);

array[0] = 1;
// array is still [4, 9]
JavaScript

In PHP, you can treat strings like arrays of characters (just like C): 

  $testString = 'mutable?';

$testString[7] = '!';
// string is now 'mutable!'
PHP

Unicode strings are tricky though. Be careful when dereferencing the array elements directly, since you'll be accessing the multi-byte strings as if it were ASCII: 

  // beware of treating multi-byte strings as mutable
$mbString = "Wełkam";

echo $mbString[2];
// echoes �  (incomplete Unicode byte sequence)

$mbString[2] = 'l';
// string is now 'Wel�kam'
PHP

Because of the complexities around ASCII and Unicode strings, the preferred practice is to treat all strings as immutable.

Mutable objects are nice because you can make changes in-place, without allocating a new object. But be careful—whenever you make an in-place change to an object, all references to that object will now reflect the change.
